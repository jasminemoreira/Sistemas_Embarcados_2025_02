Tecnologias

Famílias ARM Cortex-M

Detalhamento das séries M0, M3, M4 e M7

🏃 Cortex-M0/M0+

  • 32-bit, arquitetura ARMv6-M
  • Ultra baixo consumo
  • Pipeline de 3 estágios
  • Ideal para IoT e wearables
  • Custo muito baixo

⚡ Cortex-M3

  • 32-bit, arquitetura ARMv7-M
  • Pipeline de 3 estágios
  • MPU (Memory Protection Unit)
  • Maior conjunto de instruções
  • Balanceamento performance/energia

🚀 Cortex-M4

  • 32-bit, arquitetura ARMv7E-M
  • Unidade de processamento DSP
  • Instruções SIMD
  • Opcional: FPU (ponto flutuante)
  • Aplicações de áudio e controle

💪 Cortex-M7

  • 32-bit, arquitetura ARMv7E-M
  • Pipeline de 6 estágios
  • Cache de instrução e dados
  • FPU dupla precisão
  • Máxima performance da família M

Barramentos

Sistema de Barramentos

AHB, APB e matriz de barramento

🚌 AHB (Advanced High-performance Bus)

  • Barramento principal de alta velocidade
  • Conecta CPU, memória e DMA
  • Suporta transferências burst
  • Largura de 32 bits
  • Clock máximo do sistema

🔌 APB1 (Advanced Peripheral Bus 1)

  • Periféricos de baixa velocidade
  • Clock típico: até 42MHz (STM32F4)
  • UART, I2C, SPI, Timers básicos
  • Menor consumo de energia
  • Interface simplificada

⚡ APB2 (Advanced Peripheral Bus 2)

  • Periféricos de alta velocidade
  • Clock típico: até 84MHz (STM32F4)
  • GPIO, ADC, Timers avançados
  • SPI de alta velocidade
  • USART síncronos

🔄 Bus Matrix

  • Matriz de interconexão
  • Acesso simultâneo aos barramentos
  • Arbitragem de prioridades
  • Reduz gargalos de comunicação
  • Melhora performance geral

Implicações Práticas

A escolha do barramento afeta diretamente a performance do periférico. Periféricos críticos ficam no APB2, enquanto funções de suporte ficam no APB1 para economizar energia.

Periféricos

Periféricos Integrados

Componentes disponíveis no chip

📡 GPIO (General Purpose I/O)

  • Pinos configuráveis como entrada/saída
  • Push-pull ou open-drain
  • Resistores pull-up/pull-down internos
  • Velocidade configurável
  • Função alternada (AF)

📊 ADC (Analog-to-Digital Converter)

  • Resolução: 12 bits típico
  • Múltiplos canais de entrada
  • Conversão por software ou trigger
  • DMA support
  • Referência de tensão interna/externa

⚡ DAC (Digital-to-Analog Converter)

  • Resolução: 12 bits
  • Saída de tensão 0-VDD
  • Buffer de saída
  • Trigger por timer ou software
  • Geração de formas de onda

⏰ Timers

  • Contadores de 16/32 bits
  • PWM (Pulse Width Modulation)
  • Input Capture / Output Compare
  • Encoder interface
  • Trigger para outros periféricos

Clock

Sistema de Clock

Fontes de clock e gerenciamento de frequência

🔄 HSI (High Speed Internal)

  • Oscilador RC interno
  • Frequência: 16 MHz (STM32F4)
  • Rápido startup
  • Menos preciso (~1% tolerância)
  • Não requer componentes externos

💎 HSE (High Speed External)

  • Cristal ou oscilador externo
  • Frequência: 4-26 MHz
  • Alta precisão (~10-50 ppm)
  • Startup mais lento
  • Requer cristal + capacitores

🚀 PLL (Phase-Locked Loop)

  • Multiplicador de frequência
  • Entrada: HSI ou HSE
  • Saída: até 180 MHz (STM32F4)
  • Configuração de divisores
  • Máxima performance do sistema

🌙 LSI/LSE (Low Speed)

  • LSI: RC 32 kHz interno
  • LSE: Cristal 32.768 kHz externo
  • Baixo consumo
  • RTC (Real Time Clock)
  • Modos de baixo consumo

Árvore de Clock - STM32F4

HSE (8MHz) → PLL → SYSCLK (168MHz)
├── AHB (168MHz) → CPU, Memory, DMA
├── APB1 (42MHz) → UART, I2C, Timers 2-7
└── APB2 (84MHz) → GPIO, ADC, Timers 1,8-11
Configuração

Configuração Prática

RCC e configuração de clock

// Configuração típica de clock STM32F4 void SystemClock_Config(void) { RCC_OscInitTypeDef RCC_OscInitStruct = {0}; RCC_ClkInitTypeDef RCC_ClkInitStruct = {0}; // Configurar fonte de alimentação __HAL_RCC_PWR_CLK_ENABLE(); __HAL_PWR_VOLTAGESCALING_CONFIG(PWR_REGULATOR_VOLTAGE_SCALE1); // HSE + PLL configuração RCC_OscInitStruct.OscillatorType = RCC_OSCILLATORTYPE_HSE; RCC_OscInitStruct.HSEState = RCC_HSE_ON; RCC_OscInitStruct.PLL.PLLState = RCC_PLL_ON; RCC_OscInitStruct.PLL.PLLSource = RCC_PLLSOURCE_HSE; RCC_OscInitStruct.PLL.PLLM = 8; // Divisor de entrada RCC_OscInitStruct.PLL.PLLN = 336; // Multiplicador RCC_OscInitStruct.PLL.PLLP = 2; // Divisor principal RCC_OscInitStruct.PLL.PLLQ = 7; // Divisor USB HAL_RCC_OscConfig(&RCC_OscInitStruct); }

⚙️ RCC (Reset and Clock Control)

Periférico responsável por gerenciar todos os clocks do sistema, incluindo habilitação/desabilitação de periféricos e configuração de prescalers.

🔧 CubeMX Clock Configuration

Ferramenta gráfica para configurar clocks visualmente, calculando automaticamente os divisores e verificando limites máximos.

Estratégias de Clock

Performance máxima: HSE + PLL | Baixo consumo: HSI reduzido | Aplicações críticas: HSE com backup HSI | Debug: HSI (mais simples)

DMA

Direct Memory Access

Transferência de dados sem intervenção da CPU

🚀 O que é DMA?

  • Transferência direta memória-periférico
  • Reduz carga da CPU
  • Transferências em background
  • Múltiplos canais simultâneos
  • Triggers por periféricos

🔄 Tipos de Transferência

  • Memória → Memória
  • Memória → Periférico
  • Periférico → Memória
  • Circular (buffer infinito)
  • Normal (quantidade definida)

⚡ Aplicações Típicas

  • ADC → Buffer de dados
  • UART → String recebida
  • SPI → Transferência de arrays
  • DAC → Geração de sinais
  • Cópia de buffers grandes

🎯 Configuração

  • Source/Destination address
  • Data size (8/16/32 bits)
  • Priority level
  • Increment mode
  • Transfer complete interrupt

Vantagens do DMA

Libera CPU para outras tarefas, transferências mais rápidas, reduz latência de resposta, essencial para aplicações de tempo real e processamento de grandes volumes de dados.

Energia

Gerenciamento de Energia

Modos de baixo consumo e otimização energética

🔋 Run Mode

Consumo: ~30-100mA
Estado: CPU ativa, todos os periféricos disponíveis
Wake-up: N/A (já acordado)

😴 Sleep Mode

Consumo: ~10-30mA
Estado: CPU parada, periféricos ativos
Wake-up: Qualquer interrupção

🛌 Stop Mode

Consumo: ~1-10μA
Estado: Clocks parados, RAM preservada
Wake-up: EXTI, RTC, Watchdog

💤 Standby Mode

Consumo: ~1-3μA
Estado: Menor consumo, RAM perdida
Wake-up: Reset, RTC, pino wake-up

Estratégias de Otimização

Desabilitar periféricos não utilizados, reduzir frequência de clock quando possível, usar DMA para reduzir wake-ups, escolher modo de sleep apropriado para cada situação.

GPIO

Configuração de GPIO

Conceitos e estruturas de configuração

🔧 Estrutura GPIO_InitTypeDef

  • Pin: Seleção do pino (GPIO_PIN_x)
  • Mode: Entrada, saída, AF, analógico
  • Pull: Pull-up, pull-down, nenhum
  • Speed: Velocidade de transição

📊 Modos de Operação

  • INPUT: Leitura de sinal digital
  • OUTPUT_PP: Saída push-pull
  • OUTPUT_OD: Saída open-drain
  • AF_PP: Função alternada

⚡ Velocidade de GPIO

  • LOW: 8MHz - LEDs, switches
  • MEDIUM: 50MHz - Aplicações gerais
  • HIGH: 100MHz - Comunicação rápida
  • VERY_HIGH: 180MHz - Sinais críticos

🔄 Sequência de Configuração

  • 1. Habilitar clock do GPIO
  • 2. Preencher estrutura Init
  • 3. Chamar HAL_GPIO_Init()
  • 4. Usar HAL_GPIO_WritePin()

Gerenciamento de Clock GPIO

Cada porta GPIO tem seu próprio clock que deve ser habilitado antes da configuração. Isso permite economia de energia desabilitando portas não utilizadas.

Prática

Exemplo Prático GPIO

LED piscante com configuração completa

// Exemplo: LED piscante com configuração completa #include "stm32f4xx_hal.h" void SystemClock_Config(void); static void MX_GPIO_Init(void); int main(void) { // Inicialização do HAL HAL_Init(); // Configurar clock do sistema SystemClock_Config(); // Inicializar GPIO MX_GPIO_Init(); // Loop principal while (1) { HAL_GPIO_WritePin(GPIOA, GPIO_PIN_5, GPIO_PIN_SET); HAL_Delay(500); HAL_GPIO_WritePin(GPIOA, GPIO_PIN_5, GPIO_PIN_RESET); HAL_Delay(500); } } static void MX_GPIO_Init(void) { GPIO_InitTypeDef GPIO_InitStruct = {0}; // Habilitar clock do GPIOA __HAL_RCC_GPIOA_CLK_ENABLE(); // Configurar PA5 como saída GPIO_InitStruct.Pin = GPIO_PIN_5; GPIO_InitStruct.Mode = GPIO_MODE_OUTPUT_PP; GPIO_InitStruct.Pull = GPIO_NOPULL; GPIO_InitStruct.Speed = GPIO_SPEED_FREQ_LOW; HAL_GPIO_Init(GPIOA, &GPIO_InitStruct); // Estado inicial - LED apagado HAL_GPIO_WritePin(GPIOA, GPIO_PIN_5, GPIO_PIN_RESET); }

🔍 Análise do Código

1. HAL_Init(): Configura SysTick e inicializa o HAL
2. SystemClock_Config(): Define frequências dos barramentos
3. __HAL_RCC_GPIOA_CLK_ENABLE(): Habilita clock do periférico
4. GPIO_InitStruct: Estrutura de configuração do pino
5. HAL_GPIO_WritePin(): Controla estado do pino
