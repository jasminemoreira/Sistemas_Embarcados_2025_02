Conceitos

O que são Sistemas Embarcados?

Definições e características fundamentais

Sistemas embarcados são sistemas computacionais dedicados a funções específicas, integrados a sistemas maiores e projetados para operar com recursos limitados e alta confiabilidade.

🎯 Propósito Específico

Desenvolvidos para executar tarefas bem definidas, ao contrário de computadores de uso geral. Cada sistema é otimizado para sua aplicação.

⚡ Recursos Limitados

Operam com restrições de memória, processamento, energia e espaço físico, exigindo programação eficiente e otimizada.

🔒 Alta Confiabilidade

Devem funcionar continuamente por longos períodos, muitas vezes sem possibilidade de manutenção ou intervenção humana.

⏰ Tempo Real

Muitos sistemas embarcados têm requisitos de tempo real, onde respostas devem ocorrer dentro de prazos determinísticos.

Exemplos do Cotidiano

Smartphones, smart TVs, micro-ondas, ABS automotivo, marcapassos, roteadores Wi-Fi, drones, smartwatches, termostatos inteligentes...

Arquiteturas

Processadores vs Microcontroladores

Entendendo as diferenças fundamentais

🖥️ Processadores (CPU)

  • Foco no processamento de dados
  • Necessita componentes externos
  • Memória externa (RAM, ROM)
  • Interfaces separadas
  • Alto consumo de energia
  • Complexidade elevada
  • Usado em PCs, servidores

🔧 Microcontroladores (MCU)

  • Sistema completo em um chip
  • CPU + memória + periféricos
  • Memória integrada (Flash, RAM)
  • Interfaces I/O incorporadas
  • Baixo consumo de energia
  • Custo reduzido
  • Usado em sistemas embarcados

Por que Microcontroladores?

Para sistemas embarcados, microcontroladores oferecem a combinação ideal de funcionalidade, eficiência energética, custo reduzido e simplicidade de implementação.

ARM

Arquitetura ARM

Advanced RISC Machine - A base dos sistemas modernos

ARM é uma família de arquiteturas RISC (Reduced Instruction Set Computer) amplamente utilizada em sistemas embarcados devido à sua eficiência energética e versatilidade.

📊 Filosofia RISC

Conjunto de instruções simplificado e otimizado, permitindo execução mais rápida e eficiente com menor consumo de energia.

🔋 Eficiência Energética

Projetada para maximizar performance por watt, ideal para dispositivos móveis e sistemas alimentados por bateria.

🏗️ Arquitetura Load/Store

Operações aritméticas apenas entre registradores, com instruções específicas para acesso à memória.

⚙️ Pipeline de Execução

Execução paralela de instruções através de pipeline, aumentando throughput e eficiência do processador.

Cortex

Famílias ARM Cortex

Cortex-A, Cortex-R e Cortex-M

🏃 Cortex-A (Application)

Aplicação: Sistemas operacionais complexos
Exemplos: Smartphones, tablets, Single Board Computers
Características: Alto desempenho, MMU, cache

⚡ Cortex-R (Real-time)

Aplicação: Sistemas de tempo real crítico
Exemplos: Sistemas automotivos, controladores industriais
Características: Determinismo, confiabilidade

🔧 Cortex-M (Microcontroller)

Aplicação: Microcontroladores e IoT
Exemplos: Wearables, sensores, automação
Características: Baixo consumo, simples, custo-efetivo

Foco da Disciplina: Cortex-M

Utilizaremos principalmente microcontroladores baseados em Cortex-M devido à sua simplicidade, eficiência energética e ampla disponibilidade no mercado de sistemas embarcados.

Hardware

Barramentos e Periféricos

Infraestrutura de comunicação interna

🚌 Barramentos Internos

Endereços: Define localização na memória
Dados: Transporta informações
Controle: Coordena operações

🔌 Interfaces de Comunicação

UART: Comunicação serial assíncrona
SPI: Interface serial síncrona
I2C: Barramento de dois fios

⚙️ Periféricos Integrados

GPIO: Entrada/saída digital
ADC: Conversão analógico-digital
PWM: Modulação por largura de pulso

⏰ Sistemas de Temporização

Timers: Contadores programáveis
RTC: Relógio em tempo real
Watchdog: Monitor de sistema

Fontes de Clock

Sistema de clock é fundamental para sincronização. Microcontroladores ARM oferecem múltiplas fontes: oscilador interno RC, cristal externo, PLL para multiplicação de frequência e clock de baixa potência para modos sleep.

Memórias

Sistemas de Memória

RAM, ROM, EPROM, EEPROM e FLASH

💾 RAM (Volátil)

Função: Armazenamento temporário
Características: Rápida, volátil
Uso: Variáveis, stack, heap

🔒 ROM (Não-volátil)

Função: Armazenamento permanente
Características: Programada em fábrica
Uso: Bootloader, firmware básico

🔧 EPROM

Função: Programável com UV
Características: Apagável com luz UV
Uso: Desenvolvimento, prototipagem

⚡ EEPROM

Função: Programável eletricamente
Características: Apagável por byte
Uso: Configurações, calibração

💿 FLASH

Função: Armazenamento de programa
Características: Apagável por bloco
Uso: Código principal, dados

Organização Típica

Flash: 32KB-1MB para código | RAM: 4KB-256KB para dados | EEPROM: 1KB-4KB para configurações
